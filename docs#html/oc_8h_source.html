349 /*
350 There are multiple ways to walk down a level in a data tree
351 depending on what kind of node we are currently visiting.
352 The possibilities are:
353 1. current node is the data for a structure or dataset node:
354  => oc_data_ithfield -- get the data node corresponding
355  to the ith field of the structure
356 2. current node is the data for a grid node:
357  => oc_data_gridarray -- get the data node for the grid array
358  => oc_data_gridmap -- get the data node for the ith grid map
359 3. current node is the data for an array of structures
360  => oc_data_ithelement -- get the data node corresponding to
361  the i'th structure in the array
362  If the structure is scalar, then the indices
363  are ignored.
364 4. current node is the data for a sequence
365  => oc_data_ithrecord -- get the data node corresponding to
366  the i'th record in the sequence
367 
368 Note that the above only apply to compound objects. Once
369 you have the data node for an atomic types object (dimensioned
370 or scalar), you can read its contents using oc_data_read
371 or oc_data_readscalar.
372 */
373
